Este repositorio se compone de:

  - Commands, una carpeta donde añadir cualquier slash command que se quiera ejecutar en el bot. La carpeta necesita obligatoriamente un archivo vacio llamado '__init__.py', para que el script que maneja el bot,
  sepa que ahí se guardan los scripts para el slash command.
    · El bot necesita "gateway intents"  y administrator (para todos los permisos) para poder funcionar.

  - Procfile, un archivo que indica que archivo es el principal, el que hay que iniciar primero.

  - main.py, el archivo principal, ejecuta tanto el bot como los scripts en el repositorio.

  - requirements.txt, un archivo que ejecuta e instala las dependencias necesarias para que los scripts se carguen correctamente.

IMPORTANTE, este repositorio requiere de los siguientes github action secrets: DISCORD_TOKEN, TokenID ; PORT, 10000 ; KEY_CODE, KeyID

  - SI SE USA UN SERVICIO EXTERNO PARA EJECUTAR O MANTENER EL BOT, SE DEBERÁN DE PONER EN ESA PLATAFORMA. La plataforma más efectiva para activar y mantener activo al bot es: render.com
    · Si se tiliza render o cualquier otro tipo de servidor externo, poner los secrets en esa plataforma.

  - PARA MANTENER ACTIVO AL BOT: uptimerobot.com | crear un monitor http(s), e insertar el keep_alive link que el servidor externo/github. ([render.com te dará automáticamente el keep_alive link])
